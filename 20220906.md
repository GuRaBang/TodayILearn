# 01장 프로그래밍

## 1.1 프로그래밍

프로그래밍은 0과 1밖에 모르는 컴퓨터가 실행할 수 있도록 정확하고 상세하게 문제(=요구사항)를 설명하는  커뮤니케이션이다. 

따라서 *1) 요구사항을 명확히 이해하고*, *2) 복잡함을 단순하게 분해하고, 3) 자료를 정리하고, 4) 순서에 맞게 행위를 배열*해야 한다.  
<br/>
<br/>


## 1.2 프로그래밍 언어

컴퓨터가 이해하는 언어는 0,1로 이뤄진 기계어다. 

개발자는 컴퓨터에게 기계어로 직접 명령하는 대신, 사람이 이해할 수 있는 **문법**으로 구성된 **프로그래밍 언어**를 사용해 프로그램을 작성한다. 그리고 **컴파일러**나 **인터프리터**가 대신 **기계어**로 변환한다. 

즉, 프로그래밍 언어는 **구문(Syntax)**과 **의미(Semantics)**의 조합으로 표현된다. 
<br/>
<br/>


## 1.3 구문과 의미

*언어의 의미는 문맥에 있는 것이지 문법에 있는 것이 아니다.*

올바른 **문법**과 적절한 **의미**를 갖는 코드를 구성해야한다. (e.g. 문맥에 따른 적절한 어휘 선택, 순차적으로 진행되는 문장 구성 등)  

즉, 작성한 코드는 프로그래밍 언어의 문법에 부합해야 하며, 수행하고자 하는 바를 정확히 수행해야한다. 

마지막으로 정리하자면 프로그래밍의 목적은 문제 해결, 즉 요구사항 실현이다.

따라서 프로그래밍은 요구사항을 분석해서 적절한 자료구조와 함수로 변환하고 흐름을 제어하는 것이다.
<br/>

# 02장 자바스크립트

## 2.1 자바스크립트의 탄생

자바스크립트는 웹페이지의 보조적인 기능을 수행하기 위해 브라우저에서 동작하는 경량 프로그래밍 언어로 처음 등장했다. 
<br/>
<br/>


## 2.2 자바스크립트의 표준화

자바스크립트의 파생 버전인 JScript가 나왔으나 자바스크립트와 JScript는 표준화되지 않아서 크로스 브라우징 이슈(브라우저에 따라 웹페이지가 동작하지 않는 것)가 발생했다. 

자바스크립트의 파편화를 방지하고 모든 브라우저에서 동작하는 표준화된 자바스크립트의 필요성이 대두되면서 ECMAScript로 표준화되었다.
